说起React Native环境搭建，其实认真按照React Native中文社区的操作顺利的话是不会出问题的。我却是那个不幸运者，弄了少说有5个小时的时间。虽然我从事iOS开发已有1年半，却很少用Terminal。

内容

本文围绕React Native中文社区来说遇到的问题

  • 在【环境需求】里的第三步 按照说明，我并没有安装nvm，最终导致$ react-native init AwesomeProject 生成项目的时候 没有iOS android 这两个文件夹。按照提示，装上就好了。
  • 装上之后模拟器可以成功运行起来，但是出错了。提示
    Node server is running and available on the same network - run ‘npm start’ from react-native root
    原因是我的MAC 8081端口没有打开，因为 AppDelegate.m 里用到了8081端口。打开方式 lsof -Pn |grep 8081。再次运行就遇到下个错误了。
  • 具体错误信息看 stackoverflow的一位朋友。我按照他的操作之后还是有其他的错误。我在想为什么他的就解决了呢？记得爬博客的时候看到一个人在项目里package.json “react-native”: “^0.19.0”版本号回退成0.17.0就好了。我按照他的操作也该成0.17.0，但又出现了另一个错误。又看了下stackoverflow，他的版本和我的不一样nvm RNApp react-native –version
    react-native-cli: 0.1.10
    react-native: 0.18.1
    我也改成0.18.1，（其实我用的是最新版的0.19.0）再次 $npm install $react-native upgrade 居然神奇的好了。
